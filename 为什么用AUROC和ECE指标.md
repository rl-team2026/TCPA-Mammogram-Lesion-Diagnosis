  为什么用 AUROC 而不是 Accuracy/Recall？
  
  Accuracy 和 Recall 依赖分类阈值??阈值选 0.5 还是 0.3，结果完全不同。医学诊断中，没有一个"标准阈值"：筛查场景宁可误报不可漏诊（阈值低），手术决策宁可漏诊不可误报（阈值高）。
  
  AUROC 是阈值无关的??它衡量的是"随机抽一个恶性样本和一个良性样本，模型给恶性样本打分更高的概率"。AUROC=0.5 是瞎猜，AUROC=1.0 是完美排序。

  │        指标         │ 依赖阈值？ │ 对不平衡敏感？ │            医学场景适用性            │
  ├────────────────────┼────────────┼────────────────┼──────────────────────────────────────┤
  │ Accuracy           │ Y          │ Y             │ 差——95%良性时，全猜良性就有95%准确率 │
  ├────────────────────┼────────────┼────────────────┼──────────────────────────────────────┤
  │ Recall/Sensitivity │ Y          │ N              │ 中——需要选定操作点                   │
  ├────────────────────┼────────────┼────────────────┼──────────────────────────────────────┤
  │ AUROC              │ N          │ N              │ 好——衡量整体排序能力，行业标准       │
  ├────────────────────┼────────────┼────────────────┼──────────────────────────────────────┤
  │ AUPRC              │ N          │ Y              │ 好——关注恶性类（少数类）表现         │
  └────────────────────┴────────────┴────────────────┴──────────────────────────────────────┘
  
 
  这也是为什么几乎所有医学影像论文（Radiology、IEEE TMI、MICCAI）都用 AUROC 作为主指标。
  
  ECE 是什么？重要吗？
  
  ECE = Expected Calibration Error（期望校准误差）
  
  衡量的是：模型说"我有 90% 把握这是恶性"时，实际真的是恶性的概率是否接近 90%？
  
  ECE = 平均 | 模型自信度 — 实际准确率 |
  
  举个例子：如果模型对所有它给 0.8 分（80% 自信）的样本，实际上只有 60% 是恶性的??那它的 ECE 就会很高，因为它过度自信了。
  
  在医学中极其重要。一个 AUROC 很高但 ECE 也很高的模型会说"这个病灶 95% 是良性"??实际上可能只有 70% 是良性。临床医生如果相信了这个置信度，可能真的会漏诊。
  
  我们的 Full + LoRA 变体就是典型案例：AUROC=0.72（还行），但 ECE=0.19（很差，严重过度自信）。这就是为什么它在 Table 1 中被判定为不可用。
